# Data Model: Agent Review Orchestration

## WorkflowState

Represents one local Claude/Codex coordination workflow.

Fields:

- `featureId`: Stable feature identifier such as `042-agent-review-orchestration`.
- `featureName`: Human-readable feature name.
- `currentBranch`: Implementation or review branch.
- `baseBranch`: Base branch for review, usually `main`.
- `expectedCommit`: Optional expected commit SHA for review/final verification.
- `validationCommands`: Ordered validation command strings.
- `scopeConstraints`: Ordered constraints that generated prompts must preserve.
- `primaryWorktreePath`: Optional path that generated prompts must tell agents not to touch.
- `workingTreePath`: Optional current worktree path where the workflow should run.
- `results`: Ordered list of `WorkflowResult`.

## WorkflowResult

Represents a locally recorded output from a human, Claude, Codex, or another reviewer.

Fields:

- `stage`: One of `implement`, `review`, `fix`, `re-review`, `final-verification`, `human-merge-decision`.
- `agent`: Human-readable source label.
- `decision`: Optional detected decision: `Approved`, `Changes Requested`, or `Unknown`.
- `recordedAt`: ISO timestamp provided by the caller or generated by the script.
- `path`: Local run-record path under `.agent-workflow/runs/<feature-id>/`.

## PromptTemplate

Versioned Markdown file containing placeholders:

- `{{featureId}}`
- `{{featureName}}`
- `{{currentBranch}}`
- `{{baseBranch}}`
- `{{expectedCommit}}`
- `{{validationCommands}}`
- `{{scopeConstraints}}`
- `{{reviewDecisionFormat}}`
- `{{safetyRules}}`
- `{{humanOnlyCommands}}`
- `{{primaryWorktreeRule}}`

## RunRecord

Local gitignored file that stores:

- recorded agent output, or
- generated prompt output.

All run records must live below `.agent-workflow/runs/<feature-id>/`.
